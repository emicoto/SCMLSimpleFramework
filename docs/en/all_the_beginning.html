Create Your MOD Project from Scratch

1. Download the Necessary Tools

First, download and install a code editor. We recommend using Visual Studio Code, as it is powerful and beginner-friendly.

2. Create a Project Folder

Organize your MOD files by creating a dedicated project folder:

  1. Create a new folder on your computer, for example: MyFirstMod.
  2. Create two subfolders inside that folder:
    • script: for storing JavaScript files.
    • twee: for storing Twee files (for SugarCube).

3. Open the Project Folder in Visual Studio Code

Open Visual Studio Code and use the File > Open Folder option to select your project folder. This will make it easy to manage all the files in the folder.

4. Create the Core MOD Files

In the script folder, create a file named modscript.js. This file will contain the JavaScript code for your MOD.

In the twee folder, create a file named modcontent.twee. This file will contain the SugarCube content and structure for your MOD.

In the root directory, create a file named boot.json to configure the basic information for your MOD.

5. Learn Basic JavaScript Syntax

Here are some examples of basic JavaScript syntax that you will use:


    // This is a comment
    // Declare a variable
    let playerName = "Hero";

    // Define a function
    function greetPlayer(name) {
        console.log("Welcome, " + name + "!");
    }

    // Call the function
    greetPlayer(playerName);


    // Perform simple logical comparisons == for equality, != for inequality, === for strict equality
    if (playerName == "Hero") {
        console.log("You are a hero!");
    } else {
        console.log("Who are you?");
    }

    // Perform simple mathematical operations
    let num1 = 10;
    let num2 = 5;
    let sum = num1 + num2;
    console.log("The sum is: " + sum);

More JavaScript syntax and features can be found in the JavaScript Tutorial and the MDN Web Docs.

6. Understand SugarCube Twee Syntax

SugarCube Twee syntax is used to create paragraphs and structure for interactive stories. Here are some basic syntax examples: 


        :: ModId_YourTitle
        Welcome to my MOD!
        <<link "Go to the next passage" "ModId_NextPassage">><</link>>

        :: ModId_NextPassage
        This is the next passage. You can return to
        <<link "the previous passage" "ModId_YourTitle">><</link>>

After making the changes, you can use a Twee compiler or a tool like Twine to run the story.
